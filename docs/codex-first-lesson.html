第一期 Codex 教学，我决定先不教代码

这两天，我把一份 AI 学习需求问卷一条条看完了。

本来我以为，大家最想学的会是工作流、自动化、Agent，或者各种新模型。

结果不是。

最多人卡住的地方，反而特别朴素。

不是 AI 不够强，也不是工具不够多。

而是不会提需求，不知道怎么把 AI 接进自己的工作，也不知道结果不稳定的时候到底该怎么改。

所以我决定，第一期 Codex 教学，不先讲那些看起来很厉害的东西。

我想先带大家做成一件更重要的事。

让一个完全不会写代码的人，第一次真正体验到，自己一句人话，也能让 AI 帮你做出一个能用的小工具。

第一期我会讲什么？

很简单，就一件事。

用 Codex，从 0 做一个“工作任务助手”。

你只需要告诉它，你是做什么的，你现在卡在哪，你想要什么结果。

然后我们一起把这个模糊想法，变成一个真的能打开、能使用、能继续改的小网页。

这节课我不追求炫技。

我更在意的是，你上完之后会突然发现:

原来 AI 不是只能陪我聊天，它是真的可以开始帮我干活了。

这，才是我想做第一期 Codex 教学的原因。

Mac 和 Windows 怎么装

下面把最适合新手的路径和能进阶的路径分开写。安装信息按 OpenAI 官方文档于 2026年3月24日核对。

Mac 安装

最省心的路径是直接用 Codex App。官方文档写明 Codex App 支持 macOS（Apple Silicon），下载安装后登录 ChatGPT 账号或 API Key，选项目目录即可开始。

方案 A：App 版
1. 打开官方 macOS 下载页
2. 安装 Codex App
3. 打开后登录 ChatGPT 账号
4. 选择你的项目文件夹
5. 确认在 Local 模式下开始使用

如果你更想在终端里用，也可以安装 CLI。

方案 B：CLI 版
# 先确认 Node.js 已安装
node -v
npm -v

# 安装 Codex CLI
npm i -g @openai/codex

# 运行
codex

# 后续升级
npm i -g @openai/codex@latest

Windows 安装

官方推荐的最顺路径是装 Windows 版 Codex App。文档里明确给了 Microsoft Store 下载方式，也给了 winget 命令。CLI 在 Windows 上官方标注为 experimental，更适合放在 WSL 里使用。

方案 A：App 版（推荐）
1. 从 Microsoft Store 安装 Codex
   或运行：
   winget install Codex -s msstore
2. 打开 Codex 并登录
3. 先装常用开发工具：
   winget install --id Git.Git
   winget install --id OpenJS.NodeJS.LTS
   winget install --id Python.Python.3.14
4. 选择项目目录后开始使用

如果你希望走 CLI 路线，我的建议是先走 WSL。这一段是我基于官方“Windows 上 CLI 最佳体验用 WSL workspace”做的实操化整理。

方案 B：WSL + CLI（进阶）
1. 先装 WSL 和 Ubuntu
2. 在 WSL 里安装 Node.js
3. 运行：
   npm i -g @openai/codex
   codex
4. 如果要和 Windows App 共用配置，可设置：
   export CODEX_HOME=/mnt/c/Users/你的用户名/.codex
给新手的建议

Mac 直接用 App 或 CLI 都行。Windows 用户第一节课统一建议先装 App，降低环境摩擦。

你讲课时要强调

安装不是重点，第一次把一个真实需求做出来才是重点。环境部分控制在 10 分钟内。

一个常见坑

Windows 上如果 PowerShell 拦了 npm，官方文档给出的常见修复是把执行策略调成 RemoteSigned。

第一期怎么讲

我建议把它讲成一堂“成就感课”，而不是“知识点课”。讲完必须让人当场觉得自己也能做。

01 开场

先讲 Codex 和普通对话型 AI 的区别。重点不是“回答问题”，而是“读代码、改文件、跑命令”。

02 拆需求

教一个公式：我的身份 + 我要解决的问题 + 我想要的结果 + 输出形式。

03 现场实操

直接做一个“工作任务助手”网页，让学员看到从需求到页面的完整过程。

04 复盘迁移

把同一套方法映射到外贸、电商、教培、文案、电工等不同职业场景。

建议标题

零基础也能用 Codex 做出第一个工作小工具

建议时长

主讲 45 分钟 + 提问 15 分钟，刚好贴合问卷里的主流接受区间。

建议交付

送一套需求模板、首课提示词模板、以及可继续改的 HTML 成果。

第一期的案例就做这个

不要从复杂案例开始。第一期最适合做一个所有人都能理解、又能马上迁移到自己工作的案例。

案例名称：工作任务助手

输入你的行业、当前任务、希望产出的结果，页面自动整理出任务拆解、可复制提示词、执行步骤和检查清单。

为什么这个案例最好

不依赖复杂接口，不需要懂后端，成品可视化强，还能让不同岗位的人立刻替换成自己的业务场景。

你可以怎么继续用这份页面

它现在既能当课件，也能当对外预热页。你后面只要继续加报名按钮、讲师介绍、学员案例，就能直接变成招生页。
