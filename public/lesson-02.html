上手 Codex：文件夹管理

Codex 的核心用法是管理文件夹和线程，很多人忽视这个然后越用越乱。这一课，带你从根源上把 Codex 用顺手。

一、核心用法：文件夹 + 线程，把你的项目管得井井有条

这是 Codex 最重要的概念，很多人忽视它，然后发现"AI做着做着就乱了"，根源就在这。

1. 文件夹（工作区） + 线程（Thread）是什么

打开 Codex，你会看到左边有一栏，里面有文件夹和一条条对话记录。这两个东西组成了 Codex 的整个工作逻辑。

📁

文件夹 = 项目仓库

像一个一个的项目盒子，用来放代码、文件、资料。每个文件夹对应电脑上的一个真实目录。

🧵

线程（Thread）= 对话记录

每一条 Thread 是一次独立的协作过程，围绕一个明确目标展开。就像群聊里的一个具体话题帖。

为什么这样设计？避免上下文污染。上午让它写网页，下午让它改文案，晚上又让它做表格——如果都塞在一个对话里，AI会混乱，你也找不到文件在哪。在 Codex 里，每个项目一个文件夹，每个任务一条 Thread，互不干扰，随时断点续写。

2. 推荐的分类方式（仅供参考）

建议在你的电脑上建一个主文件夹叫 dev，然后在里面这样分类：

📚 Learning

存放学习资料、笔记、整理好的参考文档。

✏️ Notes

你自己写的文章、草稿、随手记的想法。

🚀 Projects

真实在做的项目，每个项目下面还可以继续分。比如"AI热点网站"就是一个项目 Thread。

🎨 Sandbox

沙盒，不知道放哪的东西先扔这，随用随清。

🛠️ Tools

你自己做好的通用脚本、可复用组件、小工具。

🤖 Bots

各种自动化机器人，比如飞书机器人、数据爬取机器人。

3. 如何把文件夹加进 Codex

在 Codex 界面里，找到"添加文件夹"的入口，把你电脑上的项目文件夹添加进来。添加后，在文件夹里开一条新的 Thread，就可以开始对话了。

一个简单规则记住就行：
· 同一个文件夹里可以做同一方向的事（共享文件、资源）
· 同一个 Thread里只推进一件具体的事
· 不同的事开不同的 Thread，这样效果最好

4. 几个必调的设置项

设置好这些，用起来才顺手：

1

保持电脑开机

在设置 → General 里，把"保持电脑开机"打开。这样 Codex 在后台跑任务时不会自动休眠。

2

Follow-up behavior → Steer

改成这个选项后，开发过程中你随时可以打断它、发新指令，不用等它完成。

3

权限改成 Full Access

在对话框首页，把权限设置成"完全访问"，这样 Codex 可以读写你的文件，不需要每次都弹窗确认。

4

输入 / 调出快捷功能

在对话框里按 / 键，可以吊起一些特殊功能，比如 Plan 模式（先规划再开发）、查看用量等。

5. 用 Plan 模式先规划再开发

这是 Codex 最重要的功能之一。每次开始做一个新项目时，先打开 Plan 模式，用嘴描述你的需求，Codex 会先帮你梳理清楚要做什么、怎么做，形成一份详细的计划文档，然后才开始动手。

为什么先用 Plan 模式？

AI 写代码和人类写作一样，大纲没对后面全错。先规划的好处是：
1. 你能提前看到 Codex 的理解对不对
2. 发现偏差可以及时纠正，不用等它做完了再返工
3. 计划文档本身也是项目记录，方便以后回顾

养成这个习惯，你做项目的成功率会大幅提升。

二、功能与配置项

Codex 有一些很棒的功能和配置项，先了解清楚再开始玩，效果更好。

1. 定时任务

定时任务就是让 Codex 在特定时间去干特定的事。比如：

举例：如果你有项目跑在云服务器上，可以设置一个每天早上9点的定时任务，让 Codex 自动巡检服务器有没有报错。如果有问题，它会自己解决，然后通过飞书机器人发消息告诉你。你完全不用管。

2. Skills 图形化管理

Skills 就是预置好的提示词模板，让 Codex 调用特定能力。相比其他工具（如 Claude Code），Codex 最大的优势是：Skills 有可视化图形界面，你不需要懂技术，就能轻松管理。

✨

查看已有 Skills

打开 Skills 界面，能清楚看到装了哪些 Skills、每个有什么用，不像其他工具那样"装了也不知道在哪"。

🛠️

Skill Creator

点击右上角"New Skill"，用嘴描述你想构建什么样的 Skill，体验跟"扣子"这类平台一样简单。

3. 全局规则（Personalization）

在"个人设置"里可以填全局规则。这个相当于给 Codex 设定"行为准则"，每次对话都会遵循。这个功能原本在命令行里叫 AGENT.md，Codex 把它可视化出来了。

新手可以直接复制这个全局规则

# Global rules for codex
## Operating principles
- Prefer small, reviewable diffs. Avoid sweeping refactors unless explicitly requested.
- Before editing, identify the file(s) to change and state the plan in 3-6 bullets.
- Never invent APIs, configs, or file paths. If unsure, search the repo first.
## Safety and secrets
- Never paste secrets, tokens, private keys, .env values, or credentials into code or logs.
## Code quality bar
- Add or update tests for behavior changes when the project has tests.
- Prefer type safety and explicit error handling.
## Output formatting
- For code changes: include a short summary + list of files changed.
## My preferences
- I like concise explanations, concrete steps, and copy-pastable commands.
- Default language for explanations: Chinese.

4. 推理深度（模型档位）

GPT-5.3-codex 有 4 个推理等级，可以理解为"档位"。档位越高，模型在给出答案前会花更多时间思考和自检，通常更稳更准确，但也更慢、更贵。

1

Low / Medium

日常简单任务，速度快，省额度。

2

High（推荐日常）

日常开发首选，兼顾速度和质量。

3

Extra High

处理复杂任务时用，比如大项目重构、疑难bug排查。速度最慢但最稳。

4

Max

最强推理模式，适合最复杂的任务。

新手建议：日常开发用 High 档就够了，真的遇到难活硬活再切换 Extra High。多出来的推理消耗在那种大活面前不值一提。
